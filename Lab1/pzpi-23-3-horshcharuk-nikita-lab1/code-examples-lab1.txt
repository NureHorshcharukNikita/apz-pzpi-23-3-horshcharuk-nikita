Мета використання:
Інструмент використано для формування текстового наповнення розділів:

3.1 Scope of Initial Release
3.2 Scope of Subsequent Releases
3.3 Limitations and Exclusions

Штучний інтелект застосовувався для структуризації опису функціональності системи Elevate, деталізації архітектурних компонентів (серверна частина, веб-клієнт, мобільний клієнт, IoT-компонент), формування опису подальших релізів із інтеграцією штучного інтелекту та визначення обмежень системи.

Внесок автора:
Автор сформулював вихідні запити з описом архітектури системи Elevate та вимог до кожного підрозділу, після чого отримані результати були перевірені на відповідність концепції системи. Згенерований текст було відредаговано, доповнено технічними деталями, уточнено термінологію, а також структуровано відповідно до вимог академічного документа. Автор здійснив фінальну стилістичну правку, узгодження розділів між собою та адаптацію змісту до загальної архітектури системи.

Використані Запити:

1. Запит для пункту 3.1 Scope of Initial Release

Запит до ШІ:
"Опиши Scope of Initial Release для системи Elevate з сервером, веб-клієнтом, мобільним клієнтом та IoT."

Результат:
Scope of Initial Release включає серверну частину як центральний компонент системи, веб-клієнт для взаємодії користувачів із платформою, мобільний клієнт для доступу до системи зі смартфонів та IoT-компонент для фізичної візуалізації досягнень. Сервер забезпечує обробку подій користувачів, нарахування балів, формування досягнень і рейтингів. Веб- та мобільний клієнти відображають прогрес користувачів і дозволяють взаємодіяти із системою. IoT-компонент відображає результати у вигляді LED-індикації та NFC/RFID взаємодії. Усі компоненти взаємодіють через сервер, формуючи єдиний цикл гейміфікації.

2. Запит для пункту 3.2 Scope of Subsequent Releases

Запит до ШІ:
"Опиши Scope of Subsequent Releases для системи Elevate з інтеграцією штучного інтелекту."

Результат:
Scope of Subsequent Releases передбачає інтеграцію штучного інтелекту для персоналізації гейміфікації та аналізу активності користувачів. AI може використовуватися для прогнозування рівня мотивації, формування рекомендацій користувачам, інтелектуальної аналітики командної роботи та динамічної адаптації стимулів. Система також передбачає збір історії подій, поведінкових показників та аналітичних метрик для навчання моделей. На початковому етапі можуть використовуватись готові AI-сервіси з подальшим донавчанням моделей на власних даних системи.

3. Запит для пункту 3.3 Limitations and Exclusions

Запит до ШІ:

Результат:
Limitations and Exclusions включають відсутність інтеграції із зовнішніми HRM і task-management системами, відсутність фінансових винагород, відсутність автоматичного контролю робочого часу, обмежене використання штучного інтелекту лише для рекомендацій, відсутність внутрішньої соціальної мережі, обмежену кастомізацію інтерфейсу, спрощену логіку IoT-компонента та підтримку роботи системи лише в межах однієї організації.
